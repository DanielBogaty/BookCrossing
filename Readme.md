Описание проекта: «BookCrossing»

1. Краткое описание (Elevator Pitch)

«BookCrossing» — это веб-платформа на PHP, созданная для сообщества буккроссеров. Сайт позволяет пользователям легко регистрировать книги, которыми они хотят поделиться, и находить интересующие книги, освобожденные другими участниками в их городе. Мы заменяем стихийные полки и записи в блокнотах удобной, централизованной цифровой системой, где связь между пользователями происходит через Telegram.

2. Цели и задачи

    Цель: Создать простую и практичную онлайн-площадку для любителей книг и буккроссинга.

    Задачи:

        Предоставить пользователям простой инструмент для регистрации и «освобождения» книг.

        Упростить процесс поиска книг.

        Обеспечить прямую связь между пользователями через ссылки на Telegram.

3. Целевая аудитория

    Активные буккроссеры.

    Любители чтения, которые хотят избавиться от прочитанных книг с пользой.

    Люди, ищующие недорогой способ найти новые интересные книги.

    Жители средних и крупных городов, где движение буккроссинга наиболее развито.

4. Функциональные возможности (Функционал)

Для неавторизованных пользователей:

    Просмотр каталога книг.

    Поиск и фильтрация книг (по названию, автору, жанру, городу).

    Регистрация и авторизация.

Для авторизованных пользователей:

    Личный кабинет:

        Редактирование профиля (аватар, информация о себе, город, ссылка на Telegram).

        Управление своими книгами.

    Работа с книгами:

        Добавление книги: Ввод названия, автора, жанра, описания, загрузка фотографии обложки.

        Смена статуса книги: Управление статусом (например, "Свободна", "Взята", "Зарезервирована").

    Интерактивность:

        Просмотр профиля другого пользователя и его ссылки на Telegram для связи.

        Система рейтинга: Пользователи могут выставлять репутацию друг другу за успешный обмен.

Для администратора:

    Панель управления всеми пользователями и книгами.

    Модерация контента.

    Управление жанрами и категориями.

5. Техническая реализация

    Серверная часть: Чистый PHP

    Клиентская часть: HTML, CSS, JavaScript.

    База данных: PostgreSQL.

    

    Безопасность:

        Хеширование паролей (использование password_hash).

        Защита от SQL-инъекций (использование Prepared Statements).

        Валидация и санация пользовательского ввода.

        Загрузка файлов с проверкой типа и размера.

6. Упрощенная структура базы данных

users (id, email, password_hash, username, avatar, city, telegram_url, rating, created_at)
books (id, user_id, title, author, description, image, status, created_at)
genres (id, name)
book_genre (book_id, genre_id)
ratings (id, from_user_id, to_user_id, rating, comment, created_at)
